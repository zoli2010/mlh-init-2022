Introduction

What is Blockchain?


Blockchain, simply put, is a chain of blocks that contains information. Blockchain can be used to make sure that digital documents are not tampered with (e.g. backdating) by giving the digital documents a timestamp. Blockchain is used to fix the double records problem without the need of a central server.

Blockchain, or the chain of blocks, is usually used for the transactions of personal items such as currency, property, contracts, and more. What makes blockchain so useful is that these transactions do not require a bank or a government. Moreover, recorded data can be very difficult to change.

Just like other technologies, blockchain is a software protocol, and it cannot be run without the usage of the Internet.

Some reasons why Blockchain technology have become so popular include resilience, time reduction, reliability, unchangeable transactions, fraud prevention, security, transparency, collaboration, and decentralization.

  • Resilence: Blockchains is often replicated architecture. The chain is still operated by most nodes in the event of a massive attack against the system.

  • Time reduction: In the financial industry, blockchain can play a vital role by allowing the quicker settlement of trades as it does not need a lengthy process of verification, settlement, and clearance because a single version of agreed-upon data of the share ledger is available between all stack holders.

  • Reliability: Blockchain certifies and verifies the identities of the interested parties. This removes double records, reducing rates and accelerates transactions.

  • Unchangeable transactions: By registering transactions in chronological order, Blockchain certifies the unalterability, of all operations which means when any new block has been added to the chain of ledgers, it cannot be removed or modified.

  • Fraud prevention: The concepts of shared information and consensus prevent possible losses due to fraud or embezzlement. In logistics-based industries, blockchain as a monitoring mechanism act to reduce costs.

  • Security: Attacking a traditional database is the bringing down of a specific target. With the help of Distributed Ledger Technology, each party holds a copy of the original chain, so the system remains operative, even the large number of other nodes fall.

  • Transparency: Changes to public blockchains are publicly viewable to everyone. This offers greater transparency, and all transactions are immutable.

  • Collaboration: Allows parties to transact directly with each other without the need for mediating third parties.

  • Decentralization: There are standards rules on how every node exchanges the blockchain information. This method ensures that all transactions are validated, and all valid transactions are added one by one.



Misconceptions


When talking about Blockchain technology, there are several misconceptions and mistakes that one can make.

For one, many people seem to think that Bitcoin is Blockchain. This is not true, as Blockchain is not actually Bitcoin. Rather, Blockchain is the technology behind Bitcoin. Bitcoin is known as a cryptocurrency, and Blockchain acts as a collection of financial accounts that tracks who owns what. Simply put, you can have Blockchain without Bitcoin, but you can't have Bitcoin without Blockchain.







Architecture

What exactly is a block?


As mentioned in the previous section, blockchain can be defined as a chain of blocks. In blockchain, a block is a information container. Blocks can store different kinds of information based on what type of block they are.

In every "chain" of blocks, the first block in the chain is known as the "Genesis" block.




Hashes


Each block has a hash. A hash is basically a string that has gone through a hashing algorithm, and is typically used for cryptography. One can think of a hash as a fingerprint or DNA: it identifies a person and it is never the same.

Hashes are useful when detecting changes to intersections. When a block is changed, the hash will change.

Each block contains data, a hash, and the hash of the previous block. In the previous section, we mentioned how blocks are difficult to change. The reason for this is because when changing a block (for instance, Block A), the hash of Block A will change, but Block A's hash stored in the next block (Block B) will not change. As such, the hash in Block A will not match the previous hash in Block B, thus invalidating the succeeding blocks.




Proof of Work


Although hashes are an excellent mechanism in cryptography to prevent attacks, modern computers are extremely advanced and are able to generate hundreds of thousands of hashes every second. Using this technology, an attacker can easily change the data of a block, and then re-generate all the hashes of the other blocks to make the blockchain valid again.

To resolve this problem, blockchain technology utilises a concept called "Proof of Work". This useful concept efficiently slows down the creation of new blocks, which prevents attackers from revalidating blockchains.

A proof of work is a computational problem that takes effort to solve. However, the time required to verify the results of the proof of work is lesser as compared to the efofrt it takes to solve the proof of work itself. For instance, a Bitcoin block takes almost 10 minutes to calculate the proof of work and add a new block to the chain.




Distributed P2P Network


When talking about Blockchain technology, there are several misconceptions and mistakes that one can make.

For one, many people seem to think that Bitcoin is Blockchain. This is not true, as Blockchain is not actually Bitcoin. Rather, Blockchain is the technology behind Bitcoin. Bitcoin is known as a cryptocurrency, and Blockchain acts as a collection of financial accounts that tracks who owns what. Simply put, you can have Blockchain without Bitcoin, but you can't have Bitcoin without Blockchain.

There are more methods in which blockchains can be safely secured. One of them is distributed P2P network.

Instead of using a central entity to manage the blockchain, it uses the distributed network, and everyone is allowed to join. When someone enters the P2P network, they will receive the entire copy of the blockchain. Each computer is also known as a "node".

When a user creates a new block, the aforementioned block is sent to all users on the P2P network. Each node needs to verify the block to make sure that it has not been maliciously altered. After a complete round of verification, each node will add the block to their blockchain.

A "consensus" is formed by all the nodes in a network. The nodes agree about what blocks are valid, and what blocks are invalid. Nodes in the network will reject blocks that are invalid and tampered with.


How to Successfully Tamper with a Blockchain


With all the security measures above, there are 3 steps to hypothetically tamper with a blockchain:

  • Tamper will all the blocks on the chain

  • Redo the proof-of-work for each block

  • Take control of greater than 50% of the P2P network

Although these 3 steps may look and sound very simple, these steps are nearly impossible to achieve.







Transactions

How does a Blockchain transaction work?


A blockchain transaction process can be simplified into 4 processes: transaction request, transaction broadcast, transaction validation, and blockchain addition.

A transaction request is made when, well, when a person requests a transaction. The transaction could be involved with cryptocurrency, contracts, records, etc.

The requested transaction is then broadcasted to a distributed P2P network.

With the help of nodes and algorithms, the transaction and the user is validated.

Once the transaction has been successfully validated, the new block is then securely added to the existing blockchain.







Versions

Blockchain 1.0: Currency


The implementation of DLT (distributed ledger technology) led to its first and obvious application: cryptocurrencies. This allows financial transactions based on blockchain technology. It is used in currency and payments. Bitcoin is the most prominent example in this segment.




Blockchain 2.0: Smart Contracts


The new key concepts are Smart Contracts, small computer programs that "live" in the blockchain. They are free computer programs that execute automatically, and check conditions defined earlier like facilitation, verification or enforcement. It is used as a replacement for traditional contracts.




Blockchain 3.0: DApps


DApps is an abbreviation of decentralized application. It has their backend code running on a decentralized peer-to-peer network. A DApp can have frontend Blockchain example code and user interfaces written in any language that can make a call to its backend, like a traditional Apps.
